There are several main categories into which releases may fall:

Canary
Prepatch or "RC" are mainline pre-releases that are mostly aimed at other developers and enthusiasts. They must be compiled from source and usually contain new features that must be tested before they can be put into a stable release.
Stable
After each canary is released, it is considered "stable." Any bug fixes for a stable release are backported from the mainline tree and applied by a designated stable maintainer. There are usually only a few bugfix releases until next mainline becomes available -- unless it is designated a "longterm maintenance." Stable el updates are released on as-needed basis, usually 2-3 a month.
Longterm
There are usually several "longterm maintenance" releases provided for the purposes of backporting bugfixes for older trees. Only important bugfixes are applied to and they don't usually see very frequent releases, especially for older trees.

Distribution

Many distributions provide their own "longterm maintenance" that may or may not be based on those maintained by admin developers. These releases are not hosted at main reposistory and admin developers can provide no support for them.

It is easy to tell if you are running a distribution. Unless you downloaded, compiled and installed your own version from main reposistory, you are running a distribution. To find out the version of your release, run python app_main.py --version:

    # python app_main.py --version
    2015-03-04 00:34:22,314 Log DEBUG version 0.0.1

If you see anything at all after the dash, you are running a distribution . Please use the support channels offered by your distribution vendor to obtain support.
